# Process-management
# Introduction

Process management is a fundamental concept in Linux and an essential skill in DevOps and system administration. It involves controlling how programs (processes) are executed, monitored, prioritized, and terminated within the operating system. Every task running on a Linux system is treated as a process, each with a unique Process ID (PID) that allows it to be managed effectively.

In this lab, we explore key Linux commands used to manage processes in a real-world environment. These include starting background processes using `sleep`, viewing active processes with `ps`, filtering process information using `grep`, terminating processes using `kill`, and monitoring system performance with `top`. We also demonstrate how to adjust process priority using `nice` and `renice`.

The goal of this exercise is to build practical understanding of how Linux handles multitasking and resource allocation, which is critical for managing servers, deploying applications, and maintaining system performance in DevOps environments.
